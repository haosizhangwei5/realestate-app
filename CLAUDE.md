# realestate-app

不動産物件管理・検索アプリケーションのプロジェクトです。

## Git 運用ルール

**コードを変更するたびに、必ずGitHubへプッシュすること。**

具体的な手順：

1. 変更をステージングする
   ```bash
   git add <変更したファイル>
   ```
2. コミットメッセージを付けてコミットする
   ```bash
   git commit -m "変更内容を簡潔に説明するメッセージ"
   ```
3. GitHubへプッシュする
   ```bash
   git push origin <ブランチ名>
   ```

### コミットメッセージの規約

- 日本語または英語で簡潔に記述する
- 何を変更したか・なぜ変更したかが分かるようにする
- 例:
  - `feat: 物件検索フィルター機能を追加`
  - `fix: 物件詳細ページの画像表示バグを修正`
  - `refactor: 物件一覧コンポーネントをリファクタリング`

### ブランチ運用

- `main`: 本番環境に対応するブランチ。直接コミットは原則禁止
- `develop`: 開発用の統合ブランチ
- 機能追加・バグ修正は `feature/xxx` または `fix/xxx` ブランチで作業し、PRを通じてマージする

## 開発ガイドライン

- コメントは原則書かない。変数名・関数名で意図が伝わるように命名する
- セキュリティ脆弱性（SQLインジェクション、XSSなど）を導入しない
- 不要な抽象化・将来のための機能追加は行わない
- エラーハンドリングはシステム境界（ユーザー入力・外部API）にのみ実装する
